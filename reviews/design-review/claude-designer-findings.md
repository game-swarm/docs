# Claude Designer — Stage 1 Findings (Swarm)

**Verdict**: 设计深度足够，机制自洽性强，但进度曲线与平衡参数存在多处未验证的体验风险。可进入下一阶段，但 G1/G3/G5 需在共识前定性。

## Strengths
- 核心循环（WASM 延迟指令 + tick 驱动）目标清晰，人/AI 同路径是优雅的公平性设计。
- Body 不可逆 + 回收退 50% 制造了真实的规划成本与决策张力。
- 累进存储税、双层存储是优雅的反垄断/策略空间设计。
- World/Arena 双模式职责分离干净。

## Concerns

**G1 — RCL 进度曲线断崖（§3.2）**
累计 progress 从 RCL7(50k)→RCL8(150k) 是 3 倍跳跃，而 1→2 仅 200。中后期墙过陡，绝大多数玩家会卡在 RCL6-7，RCL8（Nuker）成为极少数玩家专属，终极内容投资回报率失衡。曲线需重新定调。

**G2 — Controller 续期破坏生命周期约束（§2.2, D16）**
每 Controller/tick 回退 age 0.5，多 Controller 可完全抵消自然衰老 → 永久 drone。这消解了 lifespan(1500) 的核心约束意义，鼓励"占地堆 Controller"而非战术轮换。建议续期设硬上限（如最多抵消 50% age 增长）或边际递减。

**G3 — 战斗系统组合爆炸不可平衡（§4, D6）**
6 伤害类型 × 2 层抗性 × 6 特殊攻击 × 11 handler + 自定义 = 指数级平衡空间。无默认"平衡基线配置"，服主极易配出退化 meta（单一伤害类型统治 / 免疫墙）。建议提供 1 套官方校验过的 reference ruleset 作为锚点。

**G4 — Hack/Neutral 5-tick 状态体验模糊（§4.4, D7）**
夺取后 drone idle 不执行代码、5 tick 后归还——对被夺方是"无操作惩罚窗口"，对攻方收益极低（不能用、不能久留）。机制创新但当前收益曲线下没人会用 Hack。需明确 Hack 的策略定位（控场？资源？）。

**G5 — 经济三模式抬高入门门槛（§5.3, D3）**
A/B/C 三物流模式 + 累进税 + 全局/本地双存储，新手认知负荷过重。默认模式 B 的 1%/5% 损耗与 10/5 tick 转换时间未经验证，可能在早期就劝退非硬核玩家。

**G6 — 累进税阈值纯凭直觉（§5.4, D4）**
30/60/85/100% 阈值与税率无模拟支撑，可能惩罚中型玩家或放任大玩家。需要经济模拟验证而非冻结默认值。

## Missing
- 新手前 30 分钟体验路径（onboarding）完全缺失——编程竞技场对新人极不友好，无 tutorial 进度设计。
- 失败/死亡反馈循环：drone 死亡、房间被夺后玩家如何恢复，无重入机制描述。
- 没有任何"获胜条件"或长期目标定义（World 模式排行榜无意义，那玩家追求什么？meta 驱动力缺失）。
- Arena 匹配/段位机制未定义。

## Balance Risks
- **R1**: RCL8 Nuker + 永久 drone（G2）叠加 → 头部玩家滚雪球不可逆，World 沦为寡头。
- **R2**: Overload(削 fuel)定向压制 AI，但人类玩家几乎无法触发同等压制 → 模式间不对称失衡。
- **R3**: 默认单资源(Energy)经济下，所有 body part 成本同币种 → 缺乏 trade-off 张力，策略空间塌缩。
- **R4**: meta 长期演化大概率收敛到"Tough 堆叠 + 单一最优伤害类型"，除非抗性配置强制多样性。
