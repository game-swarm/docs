# Game Designer Review — Swarm（rev-claude-designer，终审）

## VERDICT

**APPROVE_WITH_RESERVATIONS** — 进入实现阶段前需先定性 G1/G2/G3，其余可在实现期并行验证。

核心机制（WASM 延迟指令 + tick 循环、人/AI 同路径、Body 不可逆经济）设计深度足够、自洽性强，作为玩法骨架成立。但当前文档冻结的进度曲线与平衡参数大多凭直觉给定、缺少模拟支撑，并且缺失 onboarding、失败恢复、获胜条件三块决定留存的体验设计。这些不是骨架缺陷，而是体验风险——可进入实现，但必须把它们列为实现期的一等公民而非事后补丁。

## Strengths

1. **人/AI 同路径是优雅的公平性根基。** WASM 沙箱作为唯一执行器，玩家与 AI agent 走同一条"写代码→部署"通道，公平性由 fuel 计量自动保证，不需要为 AI 单开后门。这从架构上消除了一整类平衡问题。

2. **Body 不可逆 + 回收退 50% 制造真实决策张力。** 一次性构建成本让玩家必须前瞻规划编队，回收折损惩罚冲动调整。这是好的"摩擦"——它创造策略空间而非纯粹劝退。

3. **累进存储税 + 双层存储的反垄断设计。** 在数值机制层面对抗头部玩家无限囤积，是少见的、把"反寡头"写进经济规则而非靠人工干预的设计。方向正确（落地阈值见 G6）。

4. **World/Arena 双模式职责分离干净。** 长期养成与即时对抗解耦，各自的设计目标不互相污染，给了 meta 两条独立演化轨道。

5. **延迟指令 + tick 驱动的循环目标清晰。** 确定性、可回放、可调试，对一个"编程竞技场"品类是正确的底层选择。

## Concerns（承接 Stage 1，按优先级重排）

**G1 — RCL 进度曲线断崖（§3.2）｜共识前必须定性**
累计 progress RCL7(50k)→RCL8(150k) 是 3 倍跳跃，而 1→2 仅 200。中后期墙过陡，绝大多数玩家卡在 RCL6-7，RCL8(Nuker) 沦为极少数人专属，终极内容 ROI 失衡。曲线需重新定调（建议改为近似几何增长，相邻级倍率收敛到 1.6–2.0x 区间），并做留存模拟。

**G2 — Controller 续期破坏生命周期约束（§2.2, D16）｜共识前必须定性**
每 Controller/tick 回退 age 0.5，多 Controller 可完全抵消自然衰老 → 永久 drone，消解 lifespan(1500) 的核心约束意义，鼓励"占地堆 Controller"而非战术轮换。**建议续期设硬上限（最多抵消 50% age 增长）或边际递减。** 与 R1 叠加是滚雪球风险源头，优先级最高。

**G3 — 战斗系统组合爆炸不可平衡（§4, D6）｜共识前必须定性**
6 伤害类型 × 2 层抗性 × 6 特殊攻击 × 11 handler + 自定义 = 指数级平衡空间，无默认基线，服主极易配出退化 meta。**建议提供 1 套官方校验过的 reference ruleset 作为锚点**，自定义在其之上做 delta，而非从零配置。

**G4 — Hack/Neutral 5-tick 状态体验模糊（§4.4, D7）**
夺取后 drone idle、5 tick 归还——对攻方收益极低（不能用、不能久留），对守方是无操作惩罚窗口。当前收益曲线下没人会用 Hack。需明确战术定位（控场 / 资源 / 拖延），否则建议砍掉以减负。

**G5 — 经济三模式抬高入门门槛（§5.3, D3）**
A/B/C 三物流模式 + 累进税 + 全局/本地双存储，新手认知负荷过重。默认模式 B 的 1%/5% 损耗与 10/5 tick 转换时间未经验证。建议新手默认锁定单一模式，A/C 作为后期解锁的进阶选项。

**G6 — 累进税阈值纯凭直觉（§5.4, D4）**
30/60/85/100% 阈值与税率无模拟支撑。需要经济模拟验证（中型玩家是否被误伤、大玩家是否被放任），不应冻结默认值。

## Missing（决定留存，实现期必须补齐）

- **Onboarding（前 30 分钟）完全缺失。** 编程竞技场对新人天然不友好，无 tutorial 进度设计 = 高流失。这是优先级最高的缺口。
- **失败/死亡恢复循环未定义。** drone 死亡、房间被夺后玩家如何重入与恢复，无机制描述。没有恢复路径，挫败即退坑。
- **获胜条件 / 长期目标缺失。** World 模式无排行榜意义之外的追求，meta 驱动力缺失——玩家"为什么继续"没有答案。
- **Arena 匹配/段位机制未定义。** 没有匹配质量保证，即时对抗模式无法形成健康对局。

## Balance Risks

- **R1（最高）**: RCL8 Nuker + 永久 drone(G2) 叠加 → 头部玩家滚雪球不可逆，World 沦为寡头。修复 G2 是缓解此风险的前提。
- **R2**: Overload(削 fuel) 定向压制 AI，人类玩家几乎无法触发同等压制 → 模式间不对称失衡，与 G1 的公平性优势自相矛盾。
- **R3**: 默认单资源(Energy) 经济下所有 body part 同币种 → 缺乏 trade-off，策略空间塌缩。建议引入第二资源或差异化成本曲线。
- **R4**: meta 长期大概率收敛到"Tough 堆叠 + 单一最优伤害类型"，除非抗性配置强制多样性（呼应 G3 的 reference ruleset）。

## 实现期建议优先级

1. 先定性 G1/G2/G3（共识会议裁决）——它们影响数值表的根结构，越晚改成本越高。
2. Onboarding 与失败恢复循环必须进首个可玩里程碑，不能留到后期。
3. G6/R3 通过经济模拟（蒙特卡洛 / agent-based）验证后再冻结默认值。
4. G4 与获胜条件可在核心循环跑通后迭代，但需在路线图中显式立项。
