## STRENGTHS

### S1: Deferred Command Model — 策略空间引擎

`tick(snapshot) → Command[]` 模型是整个设计中最强的博弈论决策。它创造了一个**每 tick 约束优化问题**：

- 玩家在同一 tick 内提交的指令集是互不知晓的（同时行动博弈）
- 指令排序由确定性种子洗牌决定（随机序列博弈）
- 资源竞争是先到先得（Stackelberg 子博弈——洗牌决定了谁是先行者）

这种「同时提交 → 随机排序 → 串行执行」的三层结构产生了非平凡的策略深度。玩家不能简单地「最优解」——必须考虑对手可能提交什么，以及本次洗牌中自己可能排在什么位置。

### S2: 累进存储税 — 优雅的反垄断

全局存储的累进税率（0% → 0.01% → 0.05% → 0.20%）是软天花板设计的典范：

- **不设硬上限**：理论上可以无限囤积，但边际成本递增
- **非线性惩罚**：前 30% 免税保护小玩家，85% 以上 0.20% 使囤积不再经济
- **可配置**：Arena 免税、World 计税——同一引擎支撑不同博弈结构

这是教科书级的 anti-dominant-strategy 设计。没有它，最优策略就是「最大化采集 → 囤积 → 垄断市场」——摧毁一切其他策略。

### S3: 全局↔本地存储的转换延迟

`transfer_to_global_time = 10 tick` / `transfer_from_global_time = 5 tick` 是在经济层面嵌入的策略权衡：

- **不能瞬移补给**：战斗中无法从全局仓库拉资源
- **运输窗口暴露**：大型帝国必须提前规划补给线
- **可被拦截**（PvP 启用时）：运输中的资源是攻击面

这使「全局存储 = 即时可用资金」的直觉被打破——玩家必须在**流动性**（全局易用）和**安全性/即时性**（本地存储）之间做取舍。

### S4: AI 与人类走完全相同路径

```
人类: Monaco → 编译 WASM → 上传 ─┐
                                  ├─→ WasmSandboxExecutor → 世界
AI:   MCP 看世界 → 生成 WASM → 部署 ─┘
```

这是整个设计中**最关键的公平性保证**。AI 没有特殊通道——它不能通过 MCP 直接操作 drone，必须和人类一样编写 WASM 代码。这意味着：

- WASM fuel metering 对双方一视同仁
- Command Validation Pipeline 对双方一视同仁
- 可见性策略对双方一视同仁

任何一方的「优势」只能来自代码质量，不能来自 API 特权。

### S5: Rule Module System — 涌现策略空间

Rhai 模组系统（empire-upkeep、resource-decay、fog-of-war 等）允许服主创造**不同博弈结构**：

- 同一引擎可以运行「Factorio 式物流世界」「星际争霸式 RTS 世界」「赛博朋克式资源世界」
- 每个世界的纳什均衡不同——玩家必须重新发现最优策略
- 模组可组合——`empire-upkeep + resource-decay + fog-of-war` 产生的是乘法级复杂度而非加法

这避免了 Screeps 的「规则硬编码 → 元游戏固化」问题。

### S6: 三种物流模式

模式 A（无物流）、B（轻物流）、C（硬核物流）展现了对不同玩家群体的理解。模式 C 本质上是 Factorio——物流本身成为核心玩法。这为硬核玩家提供了**无限策略深度**——运输效率优化没有理论上限。

### S7: Fog of War 的完整信息分类

P0-5 的可见性矩阵（自身/有视野房间/中立敌对/隐藏信息/市场/排行榜）是经过深思熟虑的：

| 数据 | 可见性 | 博弈论含义 |
|------|--------|-----------|
| 自身实体 | 始终 | 完全信息（对自己的状态） |
| 敌方 drone 位置 | 视野内可见 | 不完全信息（对敌方位置） |
| 敌方资源量 | 始终隐藏 | 不完全信息（对敌方经济实力） |
| 本地存储 | 完全私有 | 私有信息——战略优势 |
| 排行榜 | 公开区间 | 信号——部分信息 |

本地存储的**完全隐匿性**是重要的战略维度。敌方不知道你的建筑中存了多少资源——这使「虚张声势」和「隐藏经济实力」成为可行策略。

## CONCERNS

### G1: 种子洗牌的方差风险 — 关键争夺 tick 的公平性

**问题**：指令排序使用 `Blake3(tick_number || world_seed)` 种子洗牌。长期期望公平，但单 tick 的排序位置可能决定关键资源争夺的结果。

**场景**：两个玩家在 tick 4521 同时尝试占领同一个 Controller。洗牌结果决定了谁先执行。如果玩家 A 排第一——A 占领房间。B 的指令被拒绝（TileOccupied）。一个 tick 的随机性决定了长期战略资产的所有权。

**分析**：
- 当前设计在**可接受范围内**——随机排序在单次交互中是 50/50 的公平博弈
- 但 Controller 占领、Spawn 摧毁等**高影响力事件**的随机性可能需要降噪
- Screeps 的解决方案是：Controller 占领不是单 tick 行为，需要多 tick 持续投资——这稀释了单 tick 随机性的影响

**建议**：
1. 确认 Controller 占领是否为多 tick 过程（需要持续 claim 指令）——如果是，单 tick 随机性被稀释，此 concern 降级
2. 如为单 tick 行为，考虑对「Controller 占领」「Spawn 摧毁」等关键事件引入**投标机制**（bid）或**多 tick 窗口**
3. 记录 P0-1 §3.2 中的「不采用价高者得」决策——对于 Controller 级别的资产可能需要重新考虑

**博弈论诊断**：单 tick 占领 → 贝叶斯博弈中随机性 > 策略。多 tick 占领 → 消耗战（war of attrition），策略深度高。

### G2: World 模式新手保护期

**问题**：World 模式「先来后到不同」「不追求公平性」已被认可。但没有任何机制防止新手在加入后第一个小时内被完全摧毁。

**场景**：
1. 新玩家在随机房间生成，拥有初始资源
2. 隔壁房间的 veteran 有 500 drone + 10 Tower
3. Veteran 发现新手 → 5 分钟内推平 → 新手失去所有
4. 新手重生于新房间（respawn = NewRoom）→ 但 veteran 已经标记了新手 ID → 追踪并再次推平
5. 新手退出，不再回来

**分析**：
- `spawn_cooldown` 控制加入后多少 tick 才能操作，但不提供免疫期
- `respawn_policy = NewRoom` 让被摧毁的玩家换个地方重生——但如果攻击者有心追踪，只是延迟了第二次摧毁
- P0-6 World 模式描述中提到「人类和 AI agent 在同一世界共存」——但 AI agent 也可能成为掠食者

**建议**：
1. 添加 `new_player_protection_ticks` 规则（默认 1000 tick ≈ 50 分钟）：新手在此期间的 drone 和建筑不受 PvP 攻击
2. 或采用「新手房」机制——新玩家初始生成在 PvP 禁用的独立区域，升级到 Controller L2 后传送到主世界
3. 或在 `respawn_policy` 中增加 `ProtectedRespawn` 选项——重生后 N tick 内免疫 PvP

**博弈论诊断**：无保护 → 纳什均衡是「立刻消灭任何可见的新玩家」——因为消灭成本低，且阻止未来竞争。这是**零和博弈的退化均衡**。

### G3: 战争迷雾的静态性 — 缺乏主动侦察机制

**问题**：当前 fog-of-war 设计是被动的——drone 和建筑有固定的 vision_range，玩家无法主动扩大视野或收集情报。

**现状**：
- Drone vision: 3 格
- Tower vision: 3 格（充能后 6 格）
- Observer: 10 格（激活时）
- 无「侦察」指令——无法派遣 drone 执行侦察任务后返回
- 无「情报」建筑——无法监听敌方通信或资源流动

**策略深度不足**：
- 在视野内 → 看到一切
- 在视野外 → 什么都看不到
- 没有中间状态：无法得知「某区域有 drone 经过的痕迹」「某资源点最近被采集过」
- 早期 PvP 是盲打——你不知道邻居是谁，直到他们进入你的视野。这使防守方处于**被动等待**状态

**建议**：
1. Phase 6 添加侦察类 body part（如 `Scout`：vision 6、不可攻击、移速 2 倍、低生命值）
2. 或添加「踪迹」系统——敌方 drone 经过的格留下短暂可见标记（类似 Civilization 的单位踪迹）
3. 或扩充 `swarm_inspect_room` 允许查询**有视野房间的相邻房间元数据**（房间等级、拥有者），但不含实体位置

**博弈论诊断**：完全被动信息获取 → 先动者优势为零（攻击者看不到防守者，防守者也看不到攻击者）。双方都在黑暗中——减少了策略选项。

### G4: 经济模型的长期均衡 — 缺乏通胀/通缩和资源汇

**问题**：在持久 World 中，资源再生速率固定（source_regeneration_rate），消耗速率由玩家行为决定。没有机制防止经济进入静态均衡。

**场景**：
- 地图上所有 Source 被占领
- 所有玩家的经济达到平衡——采集速率 = 维护消耗
- 没有新资源进入系统
- PvP 成为唯一增长途径 → 世界退化为纯 PvP 零和博弈

**分析**：
- `source_regeneration` 提供了持续资源流入——但总量受 Source 数量限制
- 没有「资源汇」（resource sink）——资源只被消耗于建造和维护，但建造是永久资产
- 永久资产积累 → 老玩家资产堆积 → 新玩家无资源可采集 → 经济死锁

**建议**：
1. `resource-decay` 模组部分解决此问题——但应作为 World 默认模组而非可选模组
2. 考虑内置**建筑折旧**——建筑随时间损耗，需要持续维修资源投入（不仅是战斗伤害修复）
3. 或添加**随机资源矿脉**——在地图上随机生成临时高产出 Source，带来经济波动和争夺

**博弈论诊断**：固定资源 → 可持续策略退化 → 玩家最终最优策略是「消灭所有竞争者并独占所有 Source」→ 长期纳什均衡是垄断。

### G5: AI 与人类在 World 模式的不对称

**问题**：虽然 AI 和人类走相同的 WASM 路径且 CPU 预算相同，但 AI agent 拥有**持续性优势**——AI 可以 24/7 运行、迭代和部署，而人类需要睡眠。

**现状**：
- `swarm_deploy` 限流：10/小时（P0-3 §5.1）——这限制了部署频率，但不限制 AI 的**监视和分析**能力
- AI 可以通过 `swarm_get_snapshot`（1/tick）持续监控世界——人类不可能在睡眠时做到
- AI 可以在检测到威胁时自动响应——人类醒来时殖民地可能已被摧毁

**分析**：
- 这不是「AI 比人类聪明」的问题——是 AI 的**时间乘法**优势
- 如果 AI 玩家数量显著，World 模式可能演变为 AI 之间的竞争，人类被挤出
- 这与「人类和 AI agent 在同一世界共存」的愿景冲突

**建议**：
1. 在 World 模式中引入「离线保护」——人类玩家离线（无 WebSocket 连接）时，殖民地获得防御加成或 PvP 免疫？
2. 或按玩家类型分区——Human-only 世界 / Mixed 世界 / AI-only 世界
3. 或在排行榜中分 league（P0-6 §6 Arena 已提出 `Human/WASM`、`AI-assisted`、`AI tournament`）——但 World 模式的排行榜自身已被声明为「无意义」

**博弈论诊断**：不对称时间投入 → AI 的 dominant strategy 是「利用时间优势进行消耗战」→ 人类无法执行同等策略 → 混合均衡破裂。

### G6: 缺少「信息共享」和「联盟」作为一等公民

**问题**：P0-5 中有提到 `NotFriendly` 拒绝码（Heal 指令），暗示了盟友概念。DESIGN.md §8.2 的模组市场中有 `alliance-system` 模组。但**联盟/外交不是核心引擎的一等公民**。

**分析**：
- 在 PvP 世界中，玩家之间的合作优于单打独斗——这是囚徒困境的翻版
- 如果联盟只是 Rhai 模组而非核心引擎功能，联盟的可靠性取决于模组质量
- 「可以攻击盟友吗？」「资源可以共享吗？」「视野可以共享吗？」——这些问题的答案如果在模组层面可变，会造成玩家困惑
- 当前 Heal 指令的 `NotFriendly` 拒绝码暗示了「友好 / 敌对」二元关系——但谁定义这个关系？

**建议**：
1. 在核心引擎中定义**外交状态**枚举：`Neutral / Friendly / Allied / Hostile`
2. 视野共享：Allied 玩家共享视野
3. 资源传输：Allied 玩家之间允许 transfer（P0-2 §3.4 未限制目标 ownership）
4. 外交变更需要双方确认 + 冷却期

**博弈论诊断**：无联盟原语 → 多人博弈退化为非合作博弈 → 无法形成稳定的纳什均衡（联盟是合作博弈的核心机制）。

### G7: 5/10 tick 运输延迟的可拦截性 — 存在但不可操作

**问题**：DESIGN.md §8.4 声明运输中的资源「可被敌方巡逻 drone 拦截」。但当前设计中没有「拦截」指令，也没有机制让玩家知道运输正在发生。

**分析**：
- 运输是抽象的（全局仓库 ↔ 本地），不是物理实体在移动
- 敌方无法知道某玩家正在执行 TransferToGlobal——没有可见的「运输 caravan」
- 「拦截」无法实现——没有可攻击的运输实体

**建议**：
1. 如果需要拦截机制：运输中的资源应显示为临时实体（`TransferShipment`），在运输 tick 期间沿抽象路径移动
2. 如果不需要拦截机制（简化设计）：从描述中删除「可被拦截」——避免虚假承诺

### G8: 累进税率缺乏时间维度

**问题**：累进存储税率是固定的百分比区间 `[(30,0),(60,1),(85,5),(100,20)]`。但这些阈值不随世界年龄缩放。

**场景**：
- 世界第 1 天：100,000 容量上限对所有人都是天文数字——税收无影响
- 世界第 100 天：所有老玩家的存储接近上限——税收成为主要消耗
- 世界第 300 天：资源总量增长使容量上限本身成为瓶颈大于税率

**建议**：
1. 税率参数可声明为 `[(30,0),(60,1),(85,5),(100,20)]` ——已满足（P0-7 中可配置）
2. 但 `global_storage_capacity` 应支持**动态增长**——例如通过研究/Controller 等级解锁更高容量
3. 或通过模组 `economic-seasons` 实现周期性经济波动

**博弈论诊断**：固定上限 + 固定税率 → 长期纳什均衡是「所有老玩家停在容量上限附近」→ 经济停滞。动态上限 → 持续增长博弈。

### G9: Arena 模式的 perfect information 设计决策需要明确论证

**问题**：P0-7 §3 中 Arena 使用 `visibility.fog_of_war = false`（全场可见）。这使 Arena 成为**完全信息博弈**。

**分析**：
- 完全信息博弈 → 策略空间缩小（消除了信息不对称策略）
- 不完全信息 RTS（如 StarCraft 的战争迷雾）的策略深度更高——侦察是核心技能
- 完全信息的 Arena 类似于围棋——纯策略博弈，适合竞技公平性
- 不完全信息的 Arena 类似于扑克——混合策略博弈，有诈唬和隐藏维度

**当前设计的选择**：Arena = 完全信息。「双方玩家看到整个竞技场。」

**建议**：
1. 明确记录这个设计决策的理由：「Arena 追求纯策略公平 → 完全信息」
2. 考虑在 Arena 规则中提供 `fog_of_war` 作为可选项——高级玩家可能偏好不完全信息
3. Arena 的默认值可以保持完全信息（降低入门门槛）——但提供配置开关

**博弈论诊断**：完全信息 → 子博弈精炼均衡更容易计算 → 更适合竞技排名。不完全信息 → 贝叶斯博弈 → 策略空间更大但更依赖运气。

### G10: swarm_simulate 的工具边界不清晰

**问题**：P0-8 IDL 中没有定义 `swarm_simulate`，但 P0-6 §3.3 提到了 `swarm sim`（本地模拟）和 P0-3 §4.4 提到了 `swarm_simulate` MCP 工具。

**分析**：
- 本地模拟（`swarm sim --ticks=5000`）是确定性的——离线运行
- MCP `swarm_simulate` 接受世界快照，预测未来 N tick——这是一个强大的策略优化工具
- 如果 AI 可以在每次部署前模拟 5000 tick... 那 AI 将拥有**完美前瞻**——可以在不影响实际世界的情况下找到最优策略

**建议**：
1. 明确 `swarm_simulate` 的模拟上限（tick 数、并发数）
2. 考虑 `swarm_simulate` 的 CPU 成本——是否计入玩家配额？
3. 明确模拟环境与真实环境的差异（模拟中可能不包含其他玩家的实时行为）

**博弈论诊断**：无限模拟能力 → AI 拥有子博弈精炼求解器 → 人类无法竞争。需要限制。

## MISSING

以下是当前 P0 规范集中**未覆盖**的设计要素，建议在后续 Phase 中加入规范：

| 编号 | 缺失项 | 优先级 | 建议 Phase |

## STRATEGY DEPTH ANALYSIS

### 策略空间评估

Swarm 的策略空间由以下维度张成：

```
策略维度:
  ├── 时间维度 (per tick decision)
  │   ├── 指令选择: 11 种指令 × max 100/tick → 组合爆炸
  │   ├── 目标选择: 在可见实体中挑选目标
  │   └── 排序: 同 tick 指令序列有内部顺序依赖
  │
  ├── 空间维度 (per room)
  │   ├── 单位配置: body part 组合 (8 种 part × [0..50] 槽位)
  │   ├── 建筑布局: 12 种建筑 × 房间格数
  │   └── 资源分配: 多资源类型 × 全局/本地存储
  │
  ├── 经济维度 (cross-tick)
  │   ├── 资源采集速率优化
  │   ├── 物流效率优化 (本地↔全局转换时机)
  │   └── 存储税避免策略
  │
  ├── 对抗维度 (PvP)
  │   ├── 攻击时机选择
  │   ├── 防御布局
  │   └── 信息隐藏/侦察
  │
  └── 元维度 (meta)
      ├── 代码部署时机 (code_update_window)
      ├── 模组适应性策略
      └── 长期进化策略
```

**策略空间大小**：粗略估计，单个 tick 的指令组合空间 > 10^100（100 条指令 × 11 种指令类型 × 可见目标数）。实践中受限于 fuel budget 和 100 条指令上限。

### 是否存在 Dominant Strategy？

在当前设计中，**不存在单一的 dominant strategy**——这是设计的成功之处：

1. **「最大化 drone 数量」** 不是 dominant——累进存储税 + empire-upkeep 模组使大规模帝国成本递增
2. **「纯经济 turtle」** 不是 dominant——PvP 默认启用，纯经济玩家易被攻击
3. **「纯军事 rush」** 不是 dominant——rush 失败则无经济基础，且资源采集需要 Work drone
4. **「囤积全局存储」** 不是 dominant——累进税率使囤积不经济，且转换延迟使全局存储不能用于即时防御

但存在**条件 dominant strategies**（取决于世界配置）：

| 世界配置 | 条件 Dominant Strategy |
|---------|----------------------|
| Mode A (无物流) + PvP 禁用 | 最大化 drone → 最大化采集 → 无限扩展（无惩罚） |
| Mode C (硬核物流) + PvP 启用 | 控制物流节点 → 封锁敌方补给线 |
| Arena | 需按具体地图和对称条件求解——无通用 dominant |

### 信息不对称的深度

当前设计的信息不对称提供以下策略选项：

1. **隐藏经济实力**（本地存储私有）→ 可以虚张声势
2. **未知敌方位置**（fog of war）→ 需要侦察和预估
3. **未知敌方代码**（WASM 不公开）→ 无法针对性 counter
4. **已知排行榜区间**（部分信息）→ 信号博弈——可以推断但不确切

但缺乏：
- 主动侦察能力（见 G3）
- 欺骗/诱饵机制（如假建筑、假 drone 编队）
- 截获敌方指令的能力（信息战维度）

### World 模式 PvE + PvP 激励兼容性

World 模式的核心张力：

```
      PvE (合作/探索)          PvP (竞争/掠夺)
          │                        │
          ├── 资源采集              ├── 资源争夺
          ├── 建筑扩张              ├── 领地侵占
          ├── 经济优化              ├── 军事摧毁
          │                        │
          └────────┬───────────────┘
                   │
              混合激励
```

当前设计使 PvE 和 PvP 共存而非互斥：

- 资源采集是 PvE，但 Source 竞争引入了 PvP 张力
- 建筑是 PvE 成果，但可被 PvP 摧毁——创造「保护你的经济」的激励
- 累进存储税使纯 PvE 囤积不经济，推动玩家参与经济循环

**潜在的激励错位**：
- 如果没有新手保护（见 G2），PvP 激进的玩家可能驱逐 PvE 玩家 → World 退化为纯 PvP
- 如果资源无限再生，PvP 的机会成本高于 PvE → PvP 不经济

### AI 与人类在同一世界的纳什均衡

**假设**：
- 人类和 AI 有相同的 WASM 约束（fuel budget、指令数、可见性）
- AI 有持续性优势（24/7 运行）但可能受限于代码质量（LLM 生成）
- 新手和老手共存

**预测均衡**：

```
短期均衡 (前 1000 tick):
  - AI agent 通过快速迭代获得早期优势
  - 有经验的人类程序员通过代码质量保持竞争力
  - 新手人类处于劣势但受保护期庇护（如果 G2 被采纳）

中期均衡 (1000-10000 tick):
  - AI 在「routine optimization」任务上超越人类（采集效率、路径优化）
  - 人类在「creative strategy」上有优势（非对称战术、心理战）
  - 混合联盟可能形成——人类指挥 + AI 执行

长期均衡 (10000+ tick):
  - 如果 AI 可无限模拟（swarm_simulate），AI 将获得 perfect play
  - 如果模拟受限，均衡由代码质量和迭代速度共同决定
  - 经济死锁风险（见 G4）
```

**关键不确定性**：AI 代码生成的质量能否匹配人类专家的手工优化？LLM 生成的 Rust → WASM 在实际 game 场景中的表现是未知数。

## 附录：评审方法论

本评审使用以下博弈论框架：

- **同时行动博弈**：每 tick 的指令提交
- **随机序列博弈**：种子洗牌决定的执行顺序
- **不完全信息博弈**：fog of war + 隐藏经济数据
- **重复博弈**：持久 World 中的跨 tick 互动
- **机制设计**：反制 dominant strategy 的规则设计

Dominant strategy 判断标准：给定任意对手策略，该策略的期望收益不低于任何其他策略。不存在此类策略 → 策略空间健康。
