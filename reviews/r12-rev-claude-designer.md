# R12 评审 — Claude Opus 4.8 / Game Designer 方向

零历史上下文评审。方向: Game Designer。聚焦平衡性、玩家体验、涌现玩法与退化策略。

## VERDICT

**REQUEST CHANGES（要求修改）**

设计核心循环（代码即军队、tick 仿真、资源/物流博弈）扎实且具备深度涌现潜力，但 §8.7 特殊攻击系统存在两处与核心哲学正面冲突的 Critical 问题，以及一处可导致不可恢复锁死的平衡漏洞。这些必须在进入实现前定义清楚，否则会破坏「胜负取决于算法思维」的根本承诺并打开 griefing 向量。Major/Minor 问题不阻塞共识，但应记录入 ROADMAP 待解。

## 阻塞项（须修复后方可收敛）

- **D1 Hack 语义未定义且与「代码即军队」冲突**：被夺取的 drone 执行谁的 WASM？部署是玩家级的，攻击者无法为单个敌方 drone 编写代码。机制自相矛盾，必须改为可实现语义（禁用/转中立/限时瘫痪）。
- **D2 Overload 死亡螺旋**：削减玩家级 fuel budget 可被叠加清零 → 对手 WASM 停摆 → 无法反制 → 继续被压制。自我强化锁死 + griefing。须设 fuel 削减下限或改为针对单 drone。

## 非阻塞但需记录

- **D3** 续期机制（claim Controller 重置全军 age）使 lifespan 形同虚设，仅惩罚龟缩新手。建议部分重置或冷却 ≥ lifespan。
- **D4** 特殊攻击命中判定无公式；Disrupt 成本/收益极不对称，可能废弃所有 channeled 攻击。需数值模型与成功率曲线。
- **D5** 同 tick 资源争用以 seeded shuffle 决胜，与「非手速非运气」承诺张力明显。须明确玩家策略能否影响仲裁，或坦承为公平随机。
- **D6** 新手断崖（改变量 → 手写编译 WASM 无过渡）。建议加「配置式 bot」缓冲层。
- **D7** 离线衰减使挂机即流失，World 模式对休闲玩家留存不友好。建议默认「休眠保护」可配置项。
- **D8** 本地存储完全隐匿削弱侦察反制，情报玩法缺可读性。建议侦察/占领揭示存储估值区间。

## STRENGTHS（设计优点）

1. **核心隐喻清晰且一致**：「代码即军队」将编程能力直接转化为博弈优势，drone 行为完全由玩家部署的 WASM 决定，这是少见的真正以算法思维为胜负轴心的设计。
2. **确定性 tick 仿真**：hash(tick, seed) 驱动的可复现仿真为公平性、回放与反作弊提供了坚实基础，且天然支持赛后分析与教学。
3. **资源/物流博弈有深度**：超线性维护费、lifespan 老化、本地存储等构成真实的经济约束，迫使玩家在扩张与自动化自治之间做权衡，具备涌现玩法空间。
4. **世界规则可配置**：伤害类型、特殊攻击、隐私分级等绑定 world.toml / body part，使社区能调出差异化模式，延长生命周期。
5. **回放隐私分级（private/allies/world/public）考虑周全**，在竞技保密与社区学习之间给出了合理光谱。
6. **特殊攻击系统的设计意图（绑定 body part、Rhai 可扩展）方向正确**——问题在于个别条目的具体语义与数值，而非系统架构本身。

——

结论：设计骨架值得推进，但 §8.7 必须先解决 D1/D2 两处 Critical。修复后建议重审 §8.7 全表以确认无新冲突。
