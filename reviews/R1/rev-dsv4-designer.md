> 评审日期：2026-06-16

## Verdict: APPROVE_WITH_RESERVATIONS

Swarm 的设计在**公平性基础设施**和**策略空间构造**两个维度上展现了一流的博弈设计思维。WASM-唯一执行路径、「种子洗牌」公平排序、可配置世界规则引擎——这三项构成了一个极其扎实的竞技编程游戏基底。然而在**长期留存**（World 模式雪球效应、缺乏终局内容）和**社交结构**（无联盟/公会系统）方面存在缺口。建议在 MVP 上线前优先处理 G1（雪球效应）、G2（World 模式激励机制）。

## Strengths

### S1: WASM-唯一路径的公平性架构（设计亮点）

```
人类：Monaco 编辑器 → 编译 WASM → 上传 ─┐
                                       ├─→ WasmSandboxExecutor → 世界
AI：  MCP 看世界 → 生成 WASM → 部署 ───┘
```

这是 Swarm 最根本的设计创新。无论代码来源是人类手写还是 AI 生成，世界只认 WASM。这意味着：
- **天然公平**：所有玩家共享 WASM fuel metering，无特权执行路径
- **无 AI 作弊**：AI agent 不能通过 MCP 直接发送 `swarm_move`、`swarm_attack` 等指令——必须写 WASM 代码
- **确定性审计**：回放验证对所有玩家一视同仁

从博弈论角度，这消除了「玩家类型」带来的不对称信息——世界不区分人类和 AI，只区分「有没有部署 WASM 模块」。这是一个**完美信息博弈**的基础设施。

### S2: 种子洗牌 (Seeded Shuffle) 公平排序

```
player_order = Blake3_XOF(tick_number || world_seed)
每 tick 随机轮换 → 长期期望均等 → 不可预测
```

解决了「相同玩家每次都在相同排序位置」的公平性问题。关键属性：
- **确定性**：相同 (tick, seed, 指令集) → 相同顺序 → 可回放
- **不可预测**：玩家无法提前知道当前 tick 的排序位置 → 无法 exploit 排序
- **长期公平**：期望均等，不偏向任何玩家

配合「先到先得」资源竞争模型，创造了有意义的策略深度：要不要多个 drone 采集同一个 Source？排在后面就浪费指令。这不是运气游戏——这是**风险计算**。

### S3: 可配置世界规则引擎的博弈灵活性

Swarm 的核心洞察——「规则是配置，不是代码」——使游戏设计从一次性的固定规则变成了可演化的生态系统。三层扩展模型（Core / Declarative / Experimental）为不同玩家群体提供了不同的博弈空间：

| Layer | 适用场景 | 博弈特性 |
|-------|---------|---------|
| Core | Vanilla 世界 | 标准化竞技，平衡性由核心团队保证 |
| Declarative | 服主自定义 | 参数调整产生新的最优策略搜索空间 |
| Experimental | 模组世界 | 开放式博弈，无已知 dominant strategy |

这使得 Swarm 既是「一个游戏」又是「无限个游戏」——类似 Minecraft 服务器生态的博弈多样性。

### S4: 后勤网络的双层架构（Controller vs Depot）

```
Controller:  领土主权 + 免费维修（容量由 RCL 决定）
Depot:       前线维修（消耗存储资源）+ 可被占领/摧毁
```

这个设计创造了非平凡的战术选择：
- **推进前线**：在敌方领地边缘建 Depot，远征 drone 不用跑回主基地
- **打击后勤**：摧毁敌方 Depot → 前线 drone 断粮 → 被迫撤退或等死
- **夺取节点**：攻占敌方 Depot 获取其资源 → 为自己前线服务

这是一个经典的**供应链攻击面**设计。从博弈论角度，谁控制了补给线谁就控制了前线——这比「谁 drone 多谁赢」的简单战力比拼深了一个维度。

### S5: 全局存储累进税 (Progressive Storage Tax)

```
0–30%:   0%（免税）
30–60%:  0.01%
60–85%:  0.05%
85–100%: 0.20%
```

这直接针对 MMO 经济中的**财富集中**问题。不加税 → 早期玩家无限囤积 → 垄断市场 → 新玩家无法进入。累进税创造了一个软上限——囤积越多成本越高。同时「本地存储隐匿性」给了囤积者一个策略出口：你可以囤，但得分散到本地仓库，这使你的供应链暴露给敌方侦察/攻击。

### S6: 特殊攻击体系的博弈多样性

8 种特殊攻击（Hack/Drain/Overload/Debilitate/Disrupt/Fortify/Leech/Fabricate）+ 6 种伤害类型 + 抗性叠加 = **非传递性博弈空间**。不存在「最佳 body part 组合」，因为 Fortify 克制 Debilitate，Disrupt 克制 Hack/Drain，EMP 抗性克制 Overload。这是剪刀石头布在 8 维空间的扩展——策略深度由反制链的长度决定。

## Concerns

### G1: World 模式雪球效应 —— 无追赶机制 (SEVERITY: HIGH)

**问题描述**：World 模式是 7×24 持久世界，玩家随时加入但起点不同。设计文档明确说「不追求公平性」。然而：

1. **新玩家 500 tick safe_mode（25 分钟）**结束后，面对的是已经运行数天/数周的世界
2. **无匹配机制**——新玩家可能与运行了 10000 tick 的老玩家相邻
3. **密度优先出生分配**只能避免被包围，不能解决「邻居比你多 100 架 drone」的问题
4. **资源点被老玩家占领**——新玩家到达时，附近的 Source 可能已被采空或占领

**博弈论分析**：这类似于一个**非对称进入博弈**，其中 incumbent（老玩家）拥有不可逾越的先发优势。如果 incumbents 可以无限扩张，新玩家的最优策略是：不加入。或者加入 → 被碾压 → 退出。

**Screeps 教训**：这正是 Screeps 的核心问题——新手区（shard0）的老玩家拥有不可撼动的地位，新玩家只能去 shard1/2/3 从零开始，而这些 shard 最终也会被占领。

**建议**：
- **赛季制 World**：每 N 周重置世界（类似 Rust 服务器的 wipe 机制），同时保留「永久世界」作为可选
- **资源再生加速区**：新玩家出生房间及相邻区域的资源再生速度翻倍，持续前 N tick
- **新手缓冲带**：世界边缘区域的 NPC 守卫或自然屏障，使新玩家有一个安全的早期发展期
- **帝国维护费非线性增长**：通过 empire-upkeep 模组的超线性系数使超大帝国不可持续，自然释放资源

### G2: World 模式缺乏激励机制 (SEVERITY: HIGH)

**问题描述**：设计文档明确说「World 模式无排行榜」——仅有「趣味展示（非竞争排名）」。这意味着 World 模式玩家除了个人成就感外，**没有外部激励**。Arena 模式有排行榜和锦标赛，但 Arena 是 P2 优先级。

**博弈论分析**：在一个没有终点的无限期博弈中，如果没有阶段性目标或外部认可，玩家效用函数退化。无限期 + 无目标 = 无聊。

**Screeps 对比**：Screeps 有 GCL（Global Control Level）作为永久进度追踪——即使殖民地被灭，GCL 保留。这是玩家的「永久身份」。Swarm 似乎缺少这种机制。

**建议**：
- **World 模式成就系统**：首次占领房间、累计采集 100 万能量、存活 10000 tick 等里程碑
- **周期性世界事件**：每 5000 tick 出现「资源潮汐」——特定资源翻倍/减半
- **GCL 等效机制**：累计 Controller progress 作为玩家永久分数，跨世界保留
- **World 模式也应考虑轻度排名**：不是「谁最强」而是「最有趣殖民地」「最古老殖民地」「最环保殖民地」等非零和指标

### G3: Overload 攻击的「反趣味」潜力 (SEVERITY: MEDIUM)

**问题描述**：Overload 减少目标 fuel budget 500k（5% of MAX_FUEL），下限为 MAX_FUEL × 0.2。即一个玩家可以被持续 Overload 到只剩 20% 的 CPU 配额。

这意味着：
- 你的 drone 代码只能执行原本 20% 的逻辑
- 你无法反击（因为需要更多指令来组织反击）
- **多个攻击者轮流 Overload 同一目标 → 目标永久处于 20% fuel 状态**

**博弈论分析**：Overload 是一个「负和」攻击——攻击者的 drone 消耗资源执行 Overload，被攻击者的效用被永久压缩。如果 Overload 的成本 < 被攻击者失去的效用，这就是一个 dominant strategy：每个人都应该 Overload 邻居来削弱他们。这可能导致 **「Overload 军备竞赛」**——游戏变成谁先 Overload 对方谁就赢。

**建议**：
- 被 Overload 的玩家在 fuel 低于某个阈值时获得**临时 Overload 抗性**（如 100 tick 免疫）
- Overload 的 fuel 削减应为临时性的——如 50 tick 后自动恢复
- 增加 Overload 的全局冷却（不限攻击者）：同一目标每 200 tick 只能被 Overload 一次
  - **注**：文档中已有「同一目标每 50 tick 最多被 Overload 一次」的规定，但 50 tick = 2.5 分钟，仍然可以在短期内被多个攻击者持续压到 20%
- 引入「Overload 反击」：被 Overload 的玩家可以消耗额外 fuel 来反弹/抵消（类似 Fortify 的特殊净化效果）

### G4: 单一资源经济（Energy-only）限制策略深度 (SEVERITY: MEDIUM)

**问题描述**：Vanilla 默认规则集只有 Energy 一种资源。虽然世界可以配置多种资源，但官方默认世界是单一资源经济。

**博弈论影响**：
- 单资源经济 → 所有决策归结为「如何最大化 Energy 收入」
- 不存在资源转换/贸易的策略深度
- Body part 最优组合可以线性规划求解
- 建筑选择变成「哪个 ROI 最高」的单一维度问题

**Screeps 对比**：Screeps 也是 Energy 为主，但后来引入了 minerals（矿物）系统，创造了新的策略维度。多资源经济（如 StarCraft 的 Minerals + Gas）天然创造更丰富的策略空间。

**建议**：
- Vanilla 世界考虑引入**第二资源**（如默认的 `Matter`），用于高级建筑和特殊攻击
- 两种资源的采集点分布在不同地形——创造「控制多种资源点」的策略需求
- 标准 RTS 的两资源模型经过数十年验证是策略深度的 sweet spot

### G5: 缺乏社交结构 / 联盟系统 (SEVERITY: MEDIUM)

**问题描述**：整个设计文档中，玩家始终是独立个体。`allied` 可见性模式存在，但没有：
- 联盟/公会/阵营的正式创建机制
- 联盟内资源共享
- 联盟领土协同防御
- 联盟 vs 联盟的大规模冲突

**博弈论分析**：在 MMO 持久世界中，联盟改变了博弈结构——从多人非合作博弈变为合作博弈。联盟内部成员可以形成「集体理性」——共享信息、协同攻击、资源互助。没有联盟系统的 World 模式是一个纯竞争博弈，缺乏合作产生的涌现行为。

**建议**：
- **轻量联盟系统**（P1）：联盟标识、共享视野、联盟聊天
- **联盟领土**（P2）：相邻联盟成员的 Controller 共享防御加成
- **联盟战争**（P2）：联盟 vs 联盟的正式宣战系统

### G6: 缺乏终局内容 / 世界事件 (SEVERITY: LOW-MEDIUM)

**问题描述**：RCL 最高 8 级，Nuker 是最终建筑。RCL 8 之后的内容空白——玩家达到天花板后无事可做。

**建议**：
- **周期性世界事件**：每 N tick 触发（如「彗星坠落」生成超大型临资源点，所有玩家争夺）
- **NPC 威胁**：中立 AI 势力定期入侵（由引擎驱动，非玩家控制）——给所有玩家一个共同敌人
- **「奇观」系统**：RCL 8 后可以建造巨型纪念碑——纯粹炫耀性消费，但提供世界级公告

### G7: drone 生命周期与玩家体验 (SEVERITY: LOW)

**问题描述**：默认 lifespan 1500 tick = 75 分钟。Controller 维修硬上限为每 tick 总 age 回退 ≤ 自然增长的 50%。这意味着：**所有 drone 最终都会死亡**。维修只能延缓，不能阻止。

**体验角度**：
- 「我精心设计的完美 drone body 组合，75 分钟后自动消失」——这可能令人沮丧
- Constant rebuild treadmill 可能从「有趣的资源管理」退化为「繁琐的维护工作」
- positive side: 防止 drone 囤积，迫使玩家持续投入经济生产

这不一定是错误的——Screeps 有相同的机制且运行良好。但应观察玩家反馈，考虑是否加入「传奇 drone」（存活超过 N tick 获得永久标记/轻微加成）。

## Strategy Depth Analysis

### 策略空间维度

| 维度 | 深度 | 说明 |
|------|------|------|
| Body Part 组合 | ★★★★☆ | 8 种 × age_modifier × cost × 能力 = 巨大组合空间 |
| 伤害/抗性体系 | ★★★★★ | 6 种伤害类型 × 组件抗性 × 属性抗性 × 免疫 = 非传递博弈 |
| 特殊攻击反制链 | ★★★★★ | Hack→Disrupt, Debilitate→Fortify, 8 种互相克制 |
| 物流网络 | ★★★★☆ | 全局/本地双层 + Controller/Depot 双节点 + 运输拦截 |
| 房间控制 | ★★★☆☆ | 5 状态机 + contested 争夺 + RCL 升级决策 |
| 经济策略 | ★★★☆☆ | World 模式下较浅（单资源），Arena 模式下较深 |
| 信息博弈 | ★★★★☆ | fog_of_war + 隐藏资源/冷却 + body part 可见 + Observer |

### Dominant Strategy 检测

在 Vanilla World 模式下（Energy-only, PvP enabled, fog_of_war=true）：

**没有发现单一 dominant strategy** —— 原因：
1. 种子洗牌打破排序优势
2. 伤害/抗性体系创造非传递性
3. 特殊攻击互克
4. 累进存储税防止囤积垄断

**但存在策略收敛风险**：
- 早期 → 快速扩张（Rush Energy → Spawn WORK+CARRY+MOVE drone → 采集 → 重复）
- 中期 → 占领 Controller → 升级 RCL → 建造 Tower 防御 → 建造 Extension 扩容
- 后期 → 军事化（Attack+Tough drone）+ 特殊攻击（Overload+Hack）

这个「rush → boom → fight」的节奏是 RTS 类型固有的，不构成设计缺陷。关键是中期到后期的过渡是否有足够的**决策分叉**——目前看来特殊攻击系统提供了这个分叉。

### 人类 vs AI 的纳什均衡

在 World 模式下，人类和 AI 共享同一世界：

1. **短期均衡**：人类和 AI 都会选择「扩张 + 防御」策略
   - 扩张太快 → 防御薄弱 → 被攻击
   - 防御太强 → 扩张慢 → 被超越
   - 最优是平衡点

2. **信息不对称**：
   - 人类不知道对面的代码是 AI 写的还是人写的
   - 但这个信息不对称**不影响最优策略**——因为你无论如何都要假设对手是理性的
   - 这正是 WASM-唯一路径的美妙之处

3. **长期均衡**：
   - 在无限期博弈中，如果没有终局条件，均衡是各方扩张到帝国维护费的平衡点
   - AI agent 可以 24/7 运行（不睡觉），人类需要离线
   - **这是一个值得关注的不对称性**——是否需要「离线保护」机制？

### 配置模组对策略空间的影响

Rhai 模组系统的一个常被忽视的游戏设计意义：**每个世界的 Nash 均衡不同**。

| 世界类型 | 均衡特性 |
|---------|---------|
| Vanilla World (empire-upkeep) | 扩张到维护费 = 收入的平衡点 |
| World with resource-decay | 囤积策略被削弱 → 更频繁的资源流动 |
| World with no fog_of_war | 完全信息博弈 → 纯策略博弈 |
| World with code_propagation_speed > 0 | 空间维度策略（代码更新的物理传播） |

这种多样性意味着 Swarm 不仅是一个游戏，而是一个**博弈实验平台**。

## 总结

Swarm 的游戏设计骨架是**一流的**。WASM-唯一公平性、种子洗牌排序、可配置世界规则、后勤网络双层架构——这些决策展现了对编程竞技游戏本质的深刻理解。

核心风险不在设计质量，而在**长期留存**。World 模式缺乏：
1. 雪球效应的反制（除了 empire-upkeep 模组）
2. 新玩家的追赶路径
3. 明确的阶段性目标和终局内容
4. 社交结构

这些风险可以通过系统性的 Season 机制、成就系统、世界事件来解决——不涉及架构改动，纯粹是游戏设计层面的补充。

**建议在 MVP 上线前优先处理 G1（雪球效应）和 G2（World 模式激励）。**
